# ES RAG BM25 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use inline execution in this session. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add Elasticsearch-backed RAG keyword retrieval and best-effort ES dual writes for new chunks.

**Architecture:** Keep PostgreSQL as source of truth and Milvus/Zilliz as vector store. Add Elasticsearch as a keyword search read model behind `RAG_KEYWORD_BACKEND`, with PostgreSQL BM25 fallback.

**Tech Stack:** FastAPI, SQLAlchemy, Celery, Elasticsearch Python client 8, PostgreSQL, Milvus/Zilliz.
